SACM API - How to start (Draft)


1. Initialize testcase

POST<domain>:<port>/api/import

To simplify the testing, this route creates a case definition and instantiates the case (this request may take long). Be aware, the imported case is incrementally changed according to the current implementation. The imported case definition is defined based on an XML schema. To customize this import please consider a local installation and modify the XML file accordingly.

2. Get logged in user

GET<domain>:<port>/api/user/me

This route returns user details of the currently logged in user. Until the CONNECARE user management is completely integrated all routes on the SACM wrapper are without access control. The wrapper injects for all requests to the underlying SocioCortex system the default user mustermann@test.sc  with the password ottto.

3. Get all cases of user

GET<domain>:<port>/api/user/:userId/cases

A list of all cases of a certain user is returned.

4. Get case details hierarchically nested

GET<domain>:<port>/api/case/:caseId/tree

This route returns all information regarding a case instance hieratically nested (not for production usage). Only permissions are not returned.

All available routes of the SACM system are documented here.
